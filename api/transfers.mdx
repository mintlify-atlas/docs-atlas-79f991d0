## GET /api/transfers

Retrieve transfer history for a wallet address with filtering and pagination support.

<Info>
  This endpoint does not require authentication for read access.
</Info>

### Query Parameters

<ParamField query="address" type="string" required>
  Wallet address to query (case-insensitive)
</ParamField>

<ParamField query="chain" type="string" default="eth">
  Blockchain network to query
</ParamField>

<ParamField query="direction" type="enum" default="all">
  Filter by transfer direction:
  - `incoming` - Only transfers received by the address
  - `outgoing` - Only transfers sent from the address
  - `all` - Both incoming and outgoing transfers
</ParamField>

<ParamField query="limit" type="number" default="50">
  Number of results per page (min: 1, max: 100)
</ParamField>

<ParamField query="cursor" type="string">
  Pagination cursor from previous response's `nextCursor`
</ParamField>

### Response

<ResponseField name="status" type="string" required>
  Returns `"ok"` on success
</ResponseField>

<ResponseField name="data" type="array" required>
  Array of transfer objects
  
  <Expandable title="Transfer object properties">
    <ResponseField name="id" type="string">
      Unique transfer ID
    </ResponseField>
    
    <ResponseField name="timestamp" type="string">
      ISO 8601 timestamp
    </ResponseField>
    
    <ResponseField name="txHash" type="string">
      Transaction hash
    </ResponseField>
    
    <ResponseField name="logIndex" type="number">
      Log index within the transaction
    </ResponseField>
    
    <ResponseField name="token" type="string">
      Token contract address
    </ResponseField>
    
    <ResponseField name="from" type="string">
      Sender address
    </ResponseField>
    
    <ResponseField name="to" type="string">
      Recipient address
    </ResponseField>
    
    <ResponseField name="amountRaw" type="string">
      Raw amount (before decimal adjustment)
    </ResponseField>
    
    <ResponseField name="amount" type="string">
      Human-readable amount (after decimal adjustment)
    </ResponseField>
    
    <ResponseField name="symbol" type="string">
      Token symbol (e.g., "USDC", "ETH")
    </ResponseField>
    
    <ResponseField name="decimals" type="number">
      Token decimals
    </ResponseField>
    
    <ResponseField name="chain" type="string">
      Blockchain network
    </ResponseField>
    
    <ResponseField name="stale" type="boolean">
      Whether this transfer data might be outdated
    </ResponseField>
    
    <ResponseField name="amountUsd" type="string | null">
      USD value of the transfer (null if price unavailable)
    </ResponseField>
    
    <ResponseField name="priceUsd" type="string | null">
      USD price per token at time of snapshot
    </ResponseField>
    
    <ResponseField name="priceTimestamp" type="string | null">
      ISO 8601 timestamp of the price snapshot
    </ResponseField>
  </Expandable>
</ResponseField>

<ResponseField name="nextCursor" type="string | null" required>
  Cursor for the next page (null if no more pages)
</ResponseField>

<ResponseField name="hasMore" type="boolean" required>
  Whether more pages are available
</ResponseField>

<ResponseField name="sync" type="object | null">
  Wallet sync status information
  
  <Expandable title="properties">
    <ResponseField name="lastSyncedBlock" type="number | null">
      Last block number synced
    </ResponseField>
    
    <ResponseField name="lastSyncedAt" type="string | null">
      ISO 8601 timestamp of last sync
    </ResponseField>
  </Expandable>
</ResponseField>

### Examples

#### Get all transfers

```bash
curl "https://your-domain.com/api/transfers?address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb&chain=eth"
```

#### Get incoming transfers only

```bash
curl "https://your-domain.com/api/transfers?address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb&direction=incoming&limit=25"
```

#### Get next page

```bash
curl "https://your-domain.com/api/transfers?address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb&cursor=cm4xyz123"
```

### Error Responses

#### Invalid Query Parameters

Status code: `400`

```json
{
  "status": "error",
  "message": "invalid_query_params",
  "issues": {
    "fieldErrors": {
      "address": ["Required"]
    }
  }
}
```

#### Internal Error

Status code: `500`

```json
{
  "status": "error",
  "message": "internal_error"
}
```

### Pagination Guide

1. Make initial request without `cursor` parameter
2. Check `hasMore` in response
3. If `hasMore` is `true`, use `nextCursor` value as `cursor` for next request
4. Repeat until `hasMore` is `false`

```bash
# Page 1
curl "https://your-domain.com/api/transfers?address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb&limit=50"

# Page 2 (using nextCursor from page 1 response)
curl "https://your-domain.com/api/transfers?address=0x742d35Cc6634C0532925a3b844Bc9e7595f0bEb&limit=50&cursor=cm4xyz123"
```
